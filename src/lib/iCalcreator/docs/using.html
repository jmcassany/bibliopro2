Description:

iCalcreator is a PHP implementation of RFC 2445 to create iCal/xCal formatted files.

1. INTRO

iCalcreator is a PHP class creating iCal formatted files, iCal/xCal, for non-calendar systems like CMS, project management systems and other applications able to produce calendar information like agendas, tasks, reports, totos, journaling data and for input to calendar systems and applications.

iCalcreator is built of a single class file with a simple interface and are calendar component property oriented.

No controls between calendar components/properties are made.

Knowledge of calendar protocol rfc2445 - Internet Calendaring and Scheduling Core Object Specification (iCalendar) AND rfc2446 - iCalendar Transport-Independent Interoperability Protocol (iTIP) Scheduling Events, BusyTime, To-dos and Journal Entries is necessary! All functions calls are made as simple as possible BUT (, !!!,) read these rfc's properly!

RFC2445 - Internet Calendaring and Scheduling Core Object Specification (iCalendar) in text and HTML format.
RFC2446 - iCalendar Transport-Independent Interoperability Protocol (iTIP) Scheduling Events, BusyTime, To-dos and Journal Entries in text format.

An example how to employ iCalcreator in software development: www.kigkonsult.se/iCalcreator/HTMLiCal/index.php.

There are iCal/xCal icons in the images directory, to use as buttons on a webpage.

3.1.3.15 validDate

Check and convert input parameter date (different formats), return date in iCalcreator (internal) date format.

format

validDate( date [, bool utc/FALSE ] )

date=int year, int month, int day [, int hour, int min, int sec, mixed tz ]]
date=array( int year, int month, int day [, int hour, int min, int sec, mixed tz ]] )
date=array ( 'year' => int year, 'month' => int month, 'day' => int day [, 'hour' => int hour, 'min' => int min, 'sec' => int sec, 'tz' => mixed tz ] )
date=array ( 'timestamp' => int timestamp [, 'tz' => mixed tz ] )
date=string datestring // date in a string, acceptable by strtotime-command, ex. '14 august 2006 16.00.00'
tz=timezone / offset
offset=(+/-)HHmm[ss]
utc=TRUE and tz = UTC offset then input local date + UTC offset returns UTC format date

example

$c = new vcalendar ();
$today = date("M d Y H:i:s");
$UTCoffset = date( 'Z'); // offset in seconds
if( '-' == substr( $UTCoffset, 0, 1 )) {
  $UTCoffset = substr( $UTCoffset, 1 );
  $sign = '-';
}
else {
  $sign = '+';
}
$UTCoffsetHour = $UTCoffset / 3600;
$UTCoffsetMod = $UTCoffset % 3600;
$UTCoffsetMin = $UTCoffsetMod / 60;
$UTCoffsetSec = $UTCoffsetMod % 60;
$UTCoffset = sprintf( $sign."%02d%02d%02d", $UTCoffsetHour, $UTCoffsetMin, $UTCoffsetSec );
$today .= ' '.$UTCoffset; // UTC offset in accepted format, (+/-)HHMMSS (or (+/-)HHMM)
$date = $c->validDate( $today );
if( FALSE !== $date ) {
  $e = new vevent();
  $e->setDtstart( $date );
  $e->createDtstart();
}
else {
  echo "unvalid date $today";
}

3.2 Calendar component object property function list

All calendar component property functions for setting data.
For property format in detail, see
RFC2445 - Internet Calendaring and Scheduling Core Object Specification (iCalendar) in text and HTML format.

"DATE WITH UTC TIME
The date with UTC time, or absolute time, is identified by a LATIN CAPITAL LETTER Z suffix character (US-ASCII decimal 90), the UTC designator, appended to the time value. For example, the following represents January 19, 1998, at 0700 UTC:

DTSTART:19980119T070000Z

The TZID property parameter MUST NOT be applied to DATE-TIME properties whose time values are specified in UTC." (quote from rfc2445)

3.2.35 TRIGGER

This property specifies when an alarm will trigger.
The default value type is DURATION. The value type can be set to a DATE-TIME value type, in which case the value MUST specify a UTC formatted DATE-TIME value.

format

setTrigger( duration [, bool relatedEnd=FALSE [, bool after=FALSE [, array xparam ]] )

setTrigger( date [, array xparam ] )

setTrigger( int year/FALSE, int month/FALSE, int day/FALSE [, int week/FALSE [, int hour/FALSE, int min/FALSE, int sec/FALSE [, bool relatedEnd=FALSE [, bool after=FALSE [, string offset/FALSE [, array xparam ]]]]]] )

duration=array ( 'week' => int week )
duration=array ( 'day' => int day [, 'hour' => int hour, 'min' => int min, 'sec' => int sec ] )
date=array ( 'year' => int year, 'month' => int month, 'day' => int day [, 'hour' => int hour, 'min' => int min, 'sec' => int sec, 'tz' => string offset ] )
date=array ( 'timestamp' => int timestamp [, 'tz' => string offset ] )
offset=(+/-)HHmm[ss], local date + UTC offset returns UTC format date
date=string datestring
// UTC date in a string, acceptable by strtotime-command, ex. '14 august 2006 16.00.00',
xparam=*[ xparamkey => xparamvalue ]

example 1

$valarm->setTrigger( FALSE, FALSE, FALSE, FALSE, 1, 2, 3, TRUE, TRUE );
// duration, 1 hour 2 min 3 sec, after end,

example 2

$valarm->setTrigger( FALSE, FALSE, FALSE, 2 );
// duration, 1 week before start

example 3

$valarm->setTrigger( array( 'year' => 2007, 'month' => 6, 'day' => 5, 'hour' => 2, 'min' => 2, 'sec' => 3, 'tz' => '-0200' ) );
// local date + UTC offset returns UTC format date

example 4

$valarm->setTrigger( array( 'year' => 2007, 'month' => 6, 'day' => 5, 'hour' => 2, 'min' => 2, 'sec' => 3 ) );
// UTC date
